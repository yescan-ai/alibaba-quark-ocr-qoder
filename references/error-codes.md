# 错误码对照表

本文档列出所有可能返回的错误码及其含义和处理建议。

## 成功码

| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| `00000` | 识别成功 | 正常返回识别结果 |

## 配置类错误

| 错误码 | 含义 | 常见原因 | 处理方式 |
|--------|------|----------|----------|
| `A0100` | API Key 未配置 | 未设置 `SCAN_WEBSERVICE_KEY` 环境变量 | 提示用户配置 API Key |
| `A0211` | 配额不足 | API 调用次数超出套餐限制 | 提示用户充值或更换 API Key |

## 输入类错误

| 错误码 | 含义 | 常见原因 | 处理方式 |
|--------|------|----------|----------|
| `A0201` | 缺少图片输入 | 未提供图片 URL、文件路径或 BASE64 数据 | 提示用户提供有效的图片输入 |
| `URL_VALIDATION_ERROR` | URL 格式不合法 | 提供的 URL 不是有效的 HTTP/HTTPS 链接 | 检查 URL 格式，确保以 `http://` 或 `https://` 开头 |
| `FILE_ERROR` | 文件读取失败 | 本地文件不存在或无法访问 | 检查文件路径是否正确，文件是否存在 |
| `FILE_READ_ERROR` | 文件读取错误 | 文件权限不足或文件被占用 | 检查文件权限，确保文件可读 |
| `BASE64_FORMAT_ERROR` | BASE64 格式错误 | 提供的 BASE64 数据格式不正确 | 检查 BASE64 数据是否为有效的 base64 编码 |
| `BASE64_DECODE_ERROR` | BASE64 解码失败 | BASE64 数据解码时出错 | 检查 BASE64 数据是否完整，是否包含非法字符 |

## 网络类错误

| 错误码 | 含义 | 常见原因 | 处理方式 |
|--------|------|----------|----------|
| `A0406` | 图片无法下载 | 图片 URL 不可访问、网络超时或服务器拒绝访问 | 检查 URL 是否可访问，或改用本地文件/BASE64 |
| `A0407` | 图片 URL 不安全 | 使用了 HTTP 而非 HTTPS | 改用 HTTPS 协议的 URL |
| `HTTP_ERROR` | HTTP 请求异常 | 网络错误、服务器错误或超时 | 稍后重试，或检查网络连接 |

## 服务类错误

| 错误码 | 含义 | 常见原因 | 处理方式 |
|--------|------|----------|----------|
| `A0300` | QPS 限制 | 短时间内请求过于频繁 | 等待几秒后重试，或降低请求频率 |
| `JSON_PARSE_ERROR` | JSON 解析失败 | API 返回的数据格式异常 | 重试请求，或联系技术支持 |

## 错误处理建议

### 自动重试场景
以下错误可以考虑自动重试（最多 2-3 次）：
- `A0300` (QPS 限制) - 等待 1-2 秒后重试
- `A0406` (图片无法下载) - 可能是临时网络问题
- `HTTP_ERROR` (HTTP 请求异常) - 可能是服务器临时故障

### 不重试场景
以下错误不应重试，应直接告知用户：
- `A0100` (API Key 未配置) - 需要用户手动配置
- `A0211` (配额不足) - 需要用户充值
- `A0201` (缺少图片输入) - 需要用户提供输入
- `URL_VALIDATION_ERROR` (URL 格式错误) - 需要用户修正 URL
- `FILE_ERROR` / `FILE_READ_ERROR` (文件错误) - 需要用户检查文件
- `BASE64_*` (BASE64 错误) - 需要用户提供正确的 BASE64 数据
- `A0407` (URL 不安全) - 需要用户改用 HTTPS

### 错误信息展示
- **A0100 特殊处理**：使用 Markdown 格式友好展示，包含配置指引和开发者后台链接
- **其他错误**：原样透出 JSON，不添加额外说明
